Chaining loot tables

lootalot allows chaining LootTable objects in order to implement more complex loot tables. There are a few use cases where you want to do this, for example gacha games where probabilties are defined by a group of items that share the same rarity instead of each item individually.

Basic chaining

To chain a loot table into another one, we can define a loot definition entry to contain a table instead of an item. If that loot definition is chosen during a loot, the parent loot table will automatically call the child one's loot function and merge the child's loot result into the parent's one: 

// Create a loot table for each of our
// desired rarity
const pool1 = new LootTable(
    [
        { item: "1★ Red Coin" },
        { item: "1★ Green Coin" },
        { item: "1★ Blue Coin" },
    ], 
);
const pool2 = new LootTable(
    [
        { item: "2★ Red Coin" },
        { item: "2★ Green Coin" },
        { item: "2★ Blue Coin" },
    ], 
);
const pool3 = new LootTable(
    [
        { item: "3★ Red Coin" },
        { item: "3★ Green Coin" },
        { item: "3★ Blue Coin" },
    ], 
);

// Create a main loot table that references
// the previous loot tables
const mainTable = new LootTable(
    [
        { table: pool1, w: 78.5 },
        { table: pool2, w: 18.5 },
        { table: pool3, w: 3 },
    ], 
);

Compositing items

If we try to define loot definitions that both have an item and a table, lootalot will try to merge that item with the result from the child table when that loot definition is picked: 

const colorTable = new LootTable(
    [
        { item: { color: "Red" } },
        { item: { color: "Green" } },
        { item: { color: "Blue" } },
    ], 
);

const mainTable = new LootTable(
    [
        { item: { rarity: 1 }, table: colorTable, w: 78.5 },
        { item: { rarity: 2 }, table: colorTable, w: 18.5 },
        { item: { rarity: 3 }, table: colorTable, w: 3 },
    ], 
);
